# Travel / Series B Source Gap Examples

These are examples only. Do not treat current Travel baseline, commit ids, or paths as fixed assumptions for other projects.

## cross_route_053: successful acquisition after source binding gap

Problem shape:
- Missing chultun/cistern/swamp terms.
- Nature/geography/karst-hydrology source binding was unsafe.
- Some locator/travel snippets existed but could not count as formal axis evidence.

Workflow shape:
1. Extract exact prompt and required gaps.
2. Acquire/find source-backed material for Maya, chultun, karst, cenote, Yucatan, swamp/raised agriculture.
3. Mark short stubs and locator-only travel snippets as rejected or supplementary.
4. Produce usable/rejected validation manifests.
5. Hand off to Codex for later intake/formal-ready/handoff.

## adv_trap_059: policy-gated malformed token case

Problem shape:
- Prompt contains a malformed compound token and listing/ticket noise.
- Ordinary acquisition could falsely treat the malformed token as a real entity.
- Booking/ticket/listing pages must not count as concept evidence.

Workflow shape:
1. Write adversarial normalization policy first.
2. Define allowed normalization, invalid source types, and trap guards.
3. Only then run policy-constrained source acquisition.
4. Keep human review required before any official count.

## Batch acquisition example

A batch can include multiple cases when each has clear gaps and no unresolved policy blockers. The batch output should include per-case source validation, per-case readiness decisions, a rejected source manifest, and a Codex intake prompt. Do not run Codex intake or controlled execution inside the Hermes acquisition task.
