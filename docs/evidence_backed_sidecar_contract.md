# Evidence-Backed Runner Sidecar Contract

This sidecar is an explicit opt-in status layer for Research/Decision runs. It is not an executor.

## Files

When enabled, `<run_dir>/evidence_backed_sidecar/` contains:

- `evidence_backed_sidecar_manifest.json`
- `evidence_backed_sidecar_validation.json`
- `evidence_backed_sidecar_status.md`

## Safety Flags

Every manifest must preserve:

- `no_auto_execution: true`
- `semantic_evaluator_used: false`
- `source_acquisition_performed_by_sidecar: false`
- `final_modified_by_sidecar: false`
- `topic_refinement_auto_executed: false`

## Supported Stages

- `status_only`
- `source_registry_gate`
- `fulltext_handoff_gate`
- `evidence_packet_gate`
- `final_traceability_gate`
- `advisory_report_gate`

The sidecar only detects existing artifacts for these stages. Missing artifacts produce warnings and a `next_action`; they are not generated by this helper.

## Runner Boundary

The runner flag is default-off. When the flag is absent, no sidecar is emitted and the main runner result contract is unchanged. When the flag is present, sidecar failures are written under the sidecar directory and do not change the main runner result.
